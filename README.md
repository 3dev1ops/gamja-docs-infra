
# Auction Platform - Docs & Infra

## 1. 프로젝트 개요
본 리포지토리는 경매 시스템을 접목한 중고거래 플랫폼 프로젝트의 핵심 설계 문서 및 통합 개발 환경(Infra) 정의를 관리한다.


## 2. 리포지토리 폴더 구조 및 내용

| 폴더/파일명 | 포함되는 주요 문서 및 파일 | 핵심 역할 |
| :--- | :--- | :--- |
| `README.md` | 현재 보고 있는 문서| 리포지토리의 진입점 및 가이드라인 |
| `01_plan/` | 프로젝트 기획서, WBS (작업 분할 구조), 시장 분석 보고서 | 프로젝트의 비전과 사업성을 정의 |
| `02_requirements/` | PRD (제품 요구사항 명세서) | 개발할 기능 및 성능 기준 정의 |
| `03_architecture/` | 시스템 아키텍처 명세서, 데이터베이스 ERD 이미지 | 4개 서비스 분할 구조와 데이터 관계를 시각적 제시 |
|`04_specification` | API 명세서 | 기술적인 통신 규약 정의 |

## 4. 문서 관리 및 컨벤션

  * 문서 작성 형식: 모든 문서는 Markdown(.md) 형식으로 작성하는 것을 원칙으로 함
  * 변경 관리: Commit Message에 변경 이유를 명확히 남겨야 함.
  * Commit Message Convention: Git Commit은 정해진 규칙에 따라 작성하여 이력을 투명하게 관리. (예: `FEAT: [Auction item] 목록 조회 로직 구현`)
